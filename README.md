**Agentic AI Middleware for Adaptive Music Scores**

N2N Bridge is a Python middleware that translates game telemetry into structured musical intent, leveraging Large Language Models (LLMs) to generate deterministic MusicXML scores in real-time. It moves beyond traditional audio-looping systems, giving composers editable, high-quality musical scaffolding based on immediate game context.

Built for the **Kanz AI Hackathon**.

## Features

* **Game State Ingestion:** Reads live game data via JSON.
* **Cognitive Bridge:** Uses the Groq API (Llama-3.3-70B) to map game parameters to musical parameters (tempo, key, instrumentation).
* **MusicXML Generation:** Outputs valid MusicXML 3.1 files that can be directly opened and edited in any notation software (e.g., MuseScore, Sibelius).
* **Real-time Adaptation:** Designed for dynamic, reactive game soundtracks that keep the human composer in the creative loop.

## Prerequisites

* Python 3.8+
* A Groq API Key (you can get one for free [here](https://console.groq.com/keys))
* MuseScore 4 (or any MusicXML compatible software)

## Installation

1. Clone this repository or download the files.
2. Install the necessary dependencies:
   ```bash
   pip install groq python-dotenv

(Note: Ensure there are no commas between the package names).
Configuration
For security, API keys are managed via environment variables.
 * Create a file named .env in the root directory of the project.
 * Add your Groq API key to this file:
   GROQ_API_KEY=your_actual_groq_api_key_here

Usage
 * Set the Game State: Ensure your telemetry.json file contains the current game state you wish to translate (e.g., player_health, environment, is_in_combat).
 * Run the Bridge: Execute the main Python script in your terminal to process the telemetry:
   python hf_bridge.py

 * Orchestrate: The script will process the telemetry, call the Groq API, and generate a file named output_scaffold.musicxml.
 * Refine: Open output_scaffold.musicxml in MuseScore 4. The AI has provided the harmonic and rhythmic foundation; now apply your human aesthetic to finish the composition!.
Project Structure
 * hf_bridge.py: The main Python script handling telemetry reading, API calls, and XML generation.
 * telemetry.json: Sample input file representing the game state.
 * output_scaffold.musicxml: The resulting musical score generated by the AI agent.
 * README.md: This documentation file.
